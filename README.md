# claim-severity-ml

Predicting insurance claim costs using the Allstate Claims Severity dataset. A tabular ML project covering SQL EDA, feature engineering, gradient boosting, and SHAP explanations.

---

## Problem

Given an insurance claim, predict the total payout (`loss`) in dollars. This is a regression problem — the goal is to minimize prediction error on unseen claims.

---

## Dataset

**Allstate Claims Severity** — [Kaggle competition](https://www.kaggle.com/c/allstate-claims-severity)

- 188,318 rows after dropping one malformed row
- 116 categorical features (`cat1`–`cat116`) — binary or multi-class, anonymized
- 14 continuous features (`cont1`–`cont14`) — normalized, anonymized
- Target: `loss` (claim payout in dollars, range $0.67–$121,012)

All features are anonymized. Domain interpretation is not possible.

## How to run

All commands run from the repo root.

**1. Install dependencies**
```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

**2. Download data**

Download `train.csv` from the [Kaggle competition page](https://www.kaggle.com/c/allstate-claims-severity/data) and place it in `data/raw/`.

**3. Train**
```bash
python src/train.py
```
Trains LightGBM and XGBoost, selects the best by MAE, saves artifacts to `artifacts/`.

**4. Evaluate**
```bash
python src/evaluate.py data/processed/val.parquet
```
Prints MAE/RMSE and saves `artifacts/residuals.png`.

**5. Explain**
```bash
python src/explain.py
```
Computes SHAP values and saves `artifacts/shap_summary.png` and `artifacts/shap_importance.csv`.

**6. Predict**
```bash
python src/predict.py '{"cat1": "A", "cont1": 0.5}'
```
Accepts a partial JSON payload — missing features are filled with training defaults.

**7. Run tests**
```bash
.venv/bin/pytest -q
```

## Modeling approach

- **Target transform**: trained on `log1p(loss)`, predictions inverted with `expm1` — because claim amounts are heavily right-skewed, this makes the regression problem more stable
- **Categorical encoding**: frequency encoding (each category replaced by its count in training data); unknown categories map to 0
- **Numeric nulls**: filled with training medians
- **Train/val split**: 80/20, `random_state=42`
- **Models compared**: median baseline, HistGradientBoosting, LightGBM, XGBoost (in `notebooks/02_modeling.ipynb`)
- **Production script**: `src/train.py` selects between LightGBM and XGBoost per the build plan — HistGradientBoosting is not included there

## Results

Evaluated on 37,664 validation rows after `expm1` inversion (original dollar scale).

| Model | MAE | RMSE |
|-------|-----|------|
| HistGradientBoosting | 1151.53 | 1934.26 |
| LightGBM | 1151.92 | 1934.37 |
| XGBoost | 1155.49 | 1936.01 |
| Median baseline | 1796.96 | 2998.64 |

**Saved model: LightGBM** (MAE 1151.92) — selected by `src/train.py` from LightGBM vs XGBoost.

Note: HistGradientBoosting was included in the notebook comparison but not in the training script. All three boosted models are effectively tied; the differences are within noise at default hyperparameters.

**Why boosted trees beat the baseline**: the median baseline predicts the same value (~$2,116) for every claim regardless of features. Boosted trees learn non-linear patterns across 130 features, reducing MAE by ~36%.

## SHAP findings

SHAP values explain each prediction in **log-loss space** (not raw dollars). Top features by mean absolute SHAP:

| Feature | Mean \|SHAP\| |
|---------|-------------|
| cat80 | 0.214 |
| cat101 | 0.102 |
| cont14 | 0.060 |
| cat2 | 0.060 |
| cat1 | 0.054 |

Feature importance is heavily concentrated: the top 10 features account for ~50% of total mean absolute SHAP, while 99 of 130 features have near-zero impact. `cat80` alone accounts for ~15% of total importance, suggesting a single anonymized categorical variable is the strongest driver of claim cost predictions.

Since all features are anonymized and frequency-encoded, directional interpretation isn't possible — we can say which features matter most, but not what they represent in the real world. See `artifacts/shap_summary.png` for the full summary plot.

The residuals plot (`artifacts/residuals.png`) shows the model systematically underpredicts large claims — mean residual is +$226 for claims under $10k but +$5,852 for claims over $10k. This is a known limitation of log-transform regression on heavy-tailed targets.

## Limitations

- All features are anonymized — no domain interpretation is possible
- No hyperparameter tuning — default settings only
- Frequency encoding loses within-category signal (e.g. "A" vs "B" is compressed to a count)
- SHAP explanations are in log-loss space, not raw dollars
